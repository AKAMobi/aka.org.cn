2) The Internet

2.1) Internet如何工作?

在你阅读Internet的历史之前，我想让你知道它是如何工作的
Internet是面向包的网络。也就是说传输的数据分成了包。这种原理不是什么新技术， 早在20世纪60年代就已经被应用了。那么，网络通过Internet上传数据发生了什么呢？
网络是通过特殊的计算机相连的，这种计算机我们把它叫做路由器。路由器检查数据包的去向，然后 决定往那儿发。当然，不是所有的路由器都跟其他的路由器相连，它们只是决定数据包的去向。
因此如果路由器知道数据的去向，必须有某一种地址。当然，就有一种地址，叫做IP - 协议。 就想我上面提到的一样，用IP的数据传输是被分成包的。这又需要另一个协议 - TCP 。
很快就会发现IP - 地址（实际上是数字）很容易被计算机处理，但不易被我们人类处理。 因此在1984年产生了域名系统。这是Internet功能的纲要， 如果你想知道更多的， 建议你去察看一下DNS和TCP/IP的术语表信息

2.2) 二十世纪60年代

你可能不相信，但网络源于60年代，甲壳虫、嬉皮士的时代，也是美苏争霸的时代。 两国修建越来越多的核武器，而且两国都拥有毁灭全世界的武力。 因此RAND公司，美国最早的冷战思想机器，面对一个战略问题。怎样使美国的权力机构在遭受 核打击之后成功通信。如果一个网络的一台计算机被毁了，通信将不可能。如果又一个中央权 力机构，他肯定是核轰炸的首要目标因此RAND发明了一种新型的网络，在1964年RAND的提议被 迅速提上日程。这份提议是Paul Baran写的，这份提议阐明这种新型的网络没有中央权力机构。除此之外，它被设计成能在不完整的状态下操作。这种网络的法则是： 所有的节点都是对等的。所有的都能发送和接受消息。所有的消息都拥抱来发送，每一个都有自己的地址。这些包会被在一个节点发送到达另一个节点。 这看起来很显然，但新的地方时数据包从网络中穿梭不重要。意思是说一个节点被破坏了，其他 的节点人就能够通信。这种过程时非常低效的，而且很慢，但非常可靠。现在Internet人就用这 种方法，到现在为止只有小小的崩溃。以这种法则建立的第一个测试网络于1968年在英国国家探索实验室投入试验。在这之后，ARPA想 在美国建立更多的用这种法则的高级网络基地。这种网络有四台高速计算机组成。1969年，第一 个节点在加利福尼亚大学洛杉矶分校(UCLA)建立。

2.3) 二十世纪70年代

1971年
 
ARPANET已有23个节点：第一个节点（1969）在UCLA，其他的节点在斯坦佛研究所，犹他大学和UCSBARPANET建成了，由于那时的计算机时间是宝贵而且昂贵的，所以ARPNET提供给科学家远程共享他们 的计算机资源。现在这好像是不可想象的，因为如今一台普通的PC就有16兆内存。这与犹他大学的计算机形成鲜明的对比。这种Honeywell 516 米逆行计算机只有12K内存！
1972年
1972年时关键的一年。BBN的Ray Tomlinson发明了第一封电子邮件。但为什么这很重要呢？ 经过多年，一个奇特的现实变得清晰了。科学家用ARPANET来互相通讯，而取代了远程计算，当然是为了共享它们的实验结果或其他东西，但最主要的是闲聊！（每个用户由他自己的电子邮件地址。） 
1973年
第一个国际节点建立。它们在英格兰和挪威。由于能用各种平台跟它连接，ARPANET逐渐成长。 （这仍旧是现在的Internet的情况）。
一年以后，Vint Cerf 和 Bob Kahn出版了"A protocol for Packet Network Internetworking" 该书指出了TCP的设计。
1976年
UUCP (Unix-to-Unix Copy) 出台。 
1979年
使用UUCP的新闻组网络系统（USENET）建立。 

2.4)二十世纪八十年代

1982年
专为ARPANET的TCP/IP协议建立。这个协议在1983年1月1日取代了NCP成为了标准。 "Internet"这个名词第一次使用。 
1983年
ARPANET 分裂成军事部分和MILNET。MILNET和上一年成立的防卫数据网融为一体。新的协议标准以及从ARPANET分列是重要的，设想最初是为军事目的建立的。
借助于TCP/IP的它的分散结构，ARPANET在80年代早期迅速成长
维斯康新大学开发了名字服务器。 
1984年
联网主机数超过1000。
引入名字服务系统(DNS)。 
1986年
国家科学基金会（NFS)想把超级计算机应用于科学研究，因此他们建立了有五台超级计算机想得的中心。开始他们想用ARPANET连接计算机，但是ARPANET的官僚作风和他们职员的短处使得NFS脱离了这种解决方案。因此他们用ARPANET的IP协议建立了他们自己的网络。NFS连接了五个中心。(56kbps).显然他们不能用这个网络连接到各个大学，因为他们没有足够的资金去建立到各个大学的电缆。 
解决办法：在同一区域的学校连成一个网络，这个网络再跟其中的一个超级计算机相连。
网络的流量稳步增长，很快，计算机和线路因大量的数据而变得很慢。 
1987年
NSF 同Merit Networks签署了一份关于提高网络效率的合同。从那时起，计算中心和线路升级了。

3.2)WWW之父 - Tim Berners - Lee

Tim BL 是WWW发展的驱动力。他在CERN工作的时候写了第一个WWW客户程序和第一个服务器程序，并且制定了一些标准，如URL、HTML和HTTP等。在这以前，他是英格兰Poole得Plessey Telecommunications的图像计算机系统的开发主任和总工程师。他又一个妻子和两个孩子。 

3.3) 超文本描述语言 - HTML

所有的WWW也都是用HTML写的。尽管某些文件有不同的扩展明（如.cfm或.asp),但它们的核心仍旧是HTML。
HTML不是像C++或Pascal那样真正的语言，它是一种描述文档的系统。WWW浏览器解释HTML -- 解码并显示它。
HTML是SGML集中在超文本上的特殊版本。（SGML用于大公司数据交换）。 HTML代码用ASCII个是写的。这又一个很好的优点，ASCII能被任何系统读入（IBM,Macintosh, UNIX, ...),这样只要浏览器在的时候WWW能被任何平台使用。
当前W3C定义的标准是HTML4。
HTML从1.0开始，它没有官方标准。HTML 1.0只有第一个流行浏览器--Mosaic能够解释。第一个HTML的官方版本是2.0。它是所有Web页的基本标准。如果你想让某一页能被任何浏览器可读，就用HTML2.0。一个更加新的、明智的标准是HTML3.2。由于不能被广泛接受HTML 3.0被精简了。
随着叠层样式表和HTML 4.0的到来，HTML回到了它真正的基础（至少W3C的本意是这样的）。生来，HTML是结构语言，而不是格式化语言。尽管存在格式文本的标记，如FONT或B,但是W3C不赞成这些。 那些不被官方标准包含的元素叫做“过时”的元素。

3.4) 浏览器

什么是浏览器?
浏览器是现实HTML代码的程序。他们用作浏览WWW,也包括FTP, USENET 或者电子邮件。 也许就是你现在用的那个。
历史 第一个浏览器(Viola, Midas)1993年一月发行了X-Window版。同时Macintosh版的浏览器发行了一个Alpha版
WWW(一个行模式浏览器)1992年一月15日通过telnet向公众开放。
第一个流行的浏览器是NCSA Mosaic。他支持HTML 1.0。（第一个Alpha版[Mosaic for X]在1993年二月发布）。它在1993年9月支持各种平台（X, PC/Windows, Macintosh)
当Marc Andreessen（Mosaic的创始人）创建了他自己的公司（Mosaic Communications Corp)(现在叫做Netscap)发布了Netscap Navigator 1.0。他很快控制了70%的浏览器市场。
微软看到这个的巨大的成功，很快免费发布了他自己的浏览器，MS Internet Explorer。
现今，有4.0的Netscape Navigator和Internet Explorer
当IE 2.0出来的时候，它确实支持一些Navigator不支持的东西，如Marquee功能（滚动文字）。另一方面，Netscape也有一些IE不支持的地方。那就是分帧功能--它允许浏览器窗口分成不同的部分和表格功能。新版本的浏览器都支持最常用的HTML元素。当然，两个浏览器都支持HTML 4标准。
