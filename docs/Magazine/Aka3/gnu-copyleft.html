什么是Copyleft?


    使一个软件自由的最简单办法就是将其公开，不加任何版权限制。 但是，这样做允许私有修改版本的存在，而它又拒绝其他人修改和重新发布的自由。 这样的私有修改版本损害了把自由给予所有用户的自由。 为比，避免此种情景，"copyleft"以一种崭新形式进行版权(copyright)限制。 典型的，copyright夺去自由；copyleft则保护自由。它是这样的一个法律声明：要求人们在传播软件时要加上使用、修改和重新分发该软件及其源代码的权力。 代码和自由成为法律上密不可分的一体。

    GNU计划所使用的copyleft是通常的copyright概念和"GNU GPL"的结合。GPL(通用公共许可证)是一个拷贝许可证，它使你具有上述的那些自由。 还有另外一个许可证声明，叫"GNU LGPL",(GNU Library General Public License) 它适用于一些GNU库的发布。 这种许可证允许在某些情况下把库联接到私有软件的可执行代码中。在GNU的每次代码发布中都包括合适的许可证。

    我们强烈鼓励你采用copyleft发布你的程序和文档，而且我们已经使你这么做尽可能简单化。关于如何应用GNU GPL的细节可以在每个许可证的结尾处得到。

　

什么是FSF?

(Free Software Foundation-自由软件基金会)

    自由软件基金会(FSF)致力于消除对计算机程序在复制、分发、理解和修改方面的限制。我们通过在所有计算领域中开发和使用自由软件——尤其是通过帮助开发一个完整的兼容Unix的集成化软件系统---"GNU"来实现这一目标。 注：GNU的读音为[guh-new], 是"GNU's Not Unix"的缩写。

    我们的名字中的"Free"一词指的是自由，而非价格。你可以通过付钱，也可以通过免费的途径得到自由软件，但无论那种，你都将得到两方面的自由：第一，复制它们并分发给你的朋友和同事的自由；第二，能够得到源码，并随意修改它们的自由。你可以研究源码以提高编程水平。也可以安装、改进它，并同其他人一起分享你的改进。如果你重新分发它们，你可以加收手续费也可以免费发放，只要你也附带软件的源码并声明遵守 GPL。

    许多组织都在分发所有可以获得的自由软件。与之不同的是，自由软件基金会致力于开发新的自由软件，以及将这些软件构造成为一个协调一致的系统，这样的系统将彻底消除使用私有软件的必要。

    除了开发GNU之外，FSF还分发GNU软件的拷贝和手册，只收取分发费用，它还接受免税的馈赠以支持GNU的开发。FSF的大多数资金来自它的软件分发服务。因此我们恳请您在可能的时候从FSF定购CD-ROM和手册（尤其是CD-ROM）。
