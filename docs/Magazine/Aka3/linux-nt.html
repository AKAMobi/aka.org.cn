Linux 如何才能灭掉 Windows NT

时间：1998-06-24

　　Linux 是 90 年代初在 Linus Torvald 领导下开发的免费软件操作系统。当时，Linus Torvald 是一名芬兰计算机系学生。Linus 出手不凡，这种 32bit 多用户和多任务 UNIX 操作系统，稳定性与灵活性兼得，用户可访问其全部源代码。

　　Linux 约在全世界范围拥有 5 百万用户，而 Windows 的各版本加起来有 2 亿用户。最近我一直关注于此。

　　Corel 公司在其 NetWinder 网络计算机上采用 Linux 作为操作系统。Sun 公司选择 Linux 作为其 UltraSPARC 平台操作系统的替换选择。

　　VAR 和系统集成商们也开始将 Linux 用于用户项目中。能控制源代码，按照需要对部件进行混合搭配，建立自定义扩展，这是使用者中意之处。

　　不过，这些都还过于初段。如果 Linux 真要与 Windows NT 过招，先得力闯三道难关。

　　1、企业质量技术支持。 对大公司来说，Linux 显得很玄。大公司认为应当从单一的联系点获取支持，而 Linux 的松散结构不太可靠。这不免使一些 IT 专业人员心生顾虑，似乎选择 Linux 就会有被解雇之忧。

　　待闯的三关中，这是最容易的一关。因障碍主要来自感觉，而这感觉并不真实。Caldera 和 Red Hat 等公司现已在全世界范围内提供友好用户商业版产品。

　　2、装载应用程序。在 Linux 产品中，必须装有先进的应用软件。在这方面，进展虽有，但太过缓慢。Corel 向 Linux 交付了 WordPerfect，并计划开发一套商务应用程序。如果没有足够的需求，销售商们不会采用 Linux。而如果没有足够的应用程序，就不会有需求。应用程序这一关攻不下来，Linux 就难以为继。

　　3、标准界面。这一关最难逾越。Linux 面临的是自身形态的问题。这在技术上并不难解决，已有公司做出了类似 Win95 界面的产品。然而，要使崇尚独立的 Linuxite 统一标准何其难也。另外，其核心组成是 Unix，对图形界面一向不以为然。

　　你是否想看到微软被斗得灰头土脸？如果 IBM 装载并支持 Linux，如果 Oracle 的所有产品都采用 Linux，如果高级销售商的组织遴选出标准 Linux 界面，建立兼容标识。那么，一切都可能发生。

　　可能吗？绝对可能。如果被迫放弃 Windows NT，微软的长期策略将全线溃败。微软打算使所有的人升级到 NT，借此大赚一笔。Linux 是否有望取而代之？要看它闯关夺隘本事了。

　　据前几天的新闻报道，IBM正在装载免费软件 Apache Web 服务器 (全球使用最广泛的 Web 服务器)，看来它装载支持 Linux 的希望不大。
