X的体系结构

  • X Server

    • 控制了所有的X资源和实体，同时负责一切由client所传来的X事件
      • X资源：色彩，字形等； 实体：输入，输出装置； X事件：键盘的输入，鼠标的移动，窗口大小改变等等； 错误信息。

  • X Client

    • 就是X中的应用程序。
      • 一个X Client若想要运行，必须打开一个显示器，连接一个X Server，然后通过与X Server之间的通信来完成所有的工作。这种结构，使得X下的软件开发非常灵活。

  • X Protocol

    • 定义了应用和它的显示之间的客户/服务器关系。
      • 通过这个协议，应用与它的显示被分离开来。隐藏了操作系统和硬件的特殊性。这样大大简化了应用的开发和可移植性。
