MiniGUI 讨论会于 1999/9/19 上午在清华大学精密仪器系
系馆如期举行。与会人员约有 20 人，其中有中国科学院
和中国科技大学、中国矿业大学等单位的老师和研究人员
参加，也有国内知名企业的代表参加。来自清华大学电子
系、自动化系的 Linux 爱好者以及 MiniGUI 的开发人员
参加了会议。会议讨论气氛热烈，达到了预期效果，取得
了重要成果。

会议内容主要如下：
    1. 现场演示了 MiniGUI 的一些基本功能特性。
    2. 由魏永明简要介绍了 MiniGUI 的开发情况，以及
今后计划进行其他开发内容。
    3. 与会人员各抒己见，讨论了各自对 MiniGUI 期望。
    4. 由 AKA 负责人何小波和陶品介绍了 AKA 的宗旨和
发展历史以及正在进行的其他自由软件项目，如由 suzhe 
发起的 Smart fdisk 项目。
    5. 由林燕峰和张劲松介绍了 KCOM Space 这一基于
组件的集成开发平台，并进行了现场演示。

就 MiniGUI 项目，与会人员达成如下共识：
    1. 与会人员肯定了 MiniGUI 的开发成果。并认为
MiniGUI 在如下领域有十分广阔的应用前景：
        a) 嵌入式系统；
        b) 工业控制；
        c) 游戏开发；
        d) 中文应用。
与会者着重讨论了 MiniGUI 在嵌入式系统的应用前景。
    2. 就 MiniGUI 在嵌入式系统中的应用，与会人员提出
了在如下方面亟待开发或改进的方面：
        a) 多平台支持和可移植性；
        b) 调试环境和仿真库；
        c) 开发工具，提供除 C 之外的其他编程接口；
        d) 工控组态软件。
    3. 另外，与会人员认为 MiniGUI 还应当在如下方面
进行开发：
        a) 基于 MiniGUI 的终端仿真或 Telnet 程序。
        b) 基于 MiniGUI 的小型浏览器。
        c) 基于 MiniGUI 的中文编辑软件，如 gvim 克隆。

综上所述，与会人员认为 MiniGUI 是一个非常具有现实意义和
应用价值的软件，但作为一个定位于系统级的软件库，MiniGUI
在外围支撑软件、应用软件、健壮性等方面还有许多需要进一
步改进的地方。尽管在国外能够找到许多不同形式的 GUI 软件，
但大部分均是商业软件，能够在国内以自由软件的方式开发一个
真正自主版权的 GUI 支持软件，其意义是不言而喻的。与会
各单位、组织及个人将在今后进行各个方面的合作，为进一步
推进 MiniGUI 的开发和完善，并在国内树立自由软件的一个
典范而精诚合作，共同发展！
