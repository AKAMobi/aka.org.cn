微软万圣节备忘录事件

软体革命第四波 - 争取软体自由权

宿命 - 开放原始码社会文化并非反微软
由于文化本质, 崇尚自由, 反独裁, 自然吸引自由人士投入
反微软 - 不过是该文化反独裁的行动之一
-- Brian Lin, 所谓的反微软暴民

微软万圣节备忘录是由微软内部所外泄的文件, 在万圣节后, 造成媒体及业界相当程度的震憾. 文件明白的指出, 微软在未来对今年来风起云涌的开放原始码界的战略, 读本文可一窥微软这家公司在对付对手的手段, 或多或少, 也可以了解到, 何以美国司法部门要控诉微软.

Linux/FreeBSD用户要看, 本文作者以微软工程师身份出身, 对Linux/FreeBSD的优劣利弊分析大多数正确 (也有不少的误解, Eric S. Raymond加以注解补正), 深入浅出. 在多方面有助于您多了解Linux/FreeBSD.

微软拥护者要看. 它以微软内部的思考出发, 与微软对外的夸大广告绝不相同, 忠诚地反应出真正地微软文化, 可让您多了解自己该要站在何处.

一般Windows用户及商业机构职员主管要看, 心理恐怖及规格污染两大战术, 是微软的主力行销技俩. 全世界的电脑用户及业者, 目前几乎都身陷其中, 成为受害者之一. 了解FUD及SS手段, 可以帮助您保护自己.

政府主管及民意代表要看. 这份文件的意涵, 代表了"美国科技统治"的势力, 欧洲/日本都尽其全力的摆脱. 台湾如果要建设科技岛的地位, 走向开放原始码, 做为国家政策, 建立属于自己的软体公共建设, 是唯一的出路. 而台湾以硬体王国的地位, 可以大量回馈给开放原始码界, 在国际间竖立优良形像. 这样的形像与贡献, 可再良性循环, 回馈给台湾硬体制造厂商, 并帮助软体支援商的成长. 最终达成科技岛的目标.

广大的中国大陆更要看. 经济要起飞, 掌握现代化的软体科技是关键要素. 从科技史来看, 购买技术绝对不等于掌握科技. 要掌握科技, 只有投入科技本身才能掌握. 开放原始码展现了一条通往科技的大道.

全文非常长, 内容亦需思索. Eric S. Raymond的注解为绿色, 译者的注解为蓝色. 请找段时间好好看, 最好是周末, 因为整份文件极为值得思索, 有些部份需要反覆看, 才能求得真意, 阅读将会相当耗时.

个人期望这些文章, 即使不能停止微软的行为(很明显), 也要多唤醒几个灵魂. 台湾电脑社会久来拜巨人脚下之风, 应当开始真正思考, 建设属于国人自己的"软体公共建设", 否则, 我们将永远挂着"硬体王国, 软体残障"的牌子, 在中正机场门前了.

内容目录:

  • 万圣节文件 - 建议您由此开始, 看完这份简介, 先跳到心理恐布战术及规格污染战术, 了解一下这两个在文件中扮演关键性角色的辞汇, 然后再看Halloween I. Halloween II及Halloween III可以另外找空暇时间再看, 这两份份量并没有那么重, 有意多了解的应当要看一看, 没时间的话, 看或不看都可以. 在Halloween I大量引用Eric S. Raymond的两篇文章, "教堂观与市集观", "开拓智域疆土", 希望深入了解的人, 则应该要研究一下.
  • Halloween I - 万圣节文件本文.
  • Halloween II - 前因后果, 尽在于此. 本文主题是Linux竞争力分析, 我将本文为看成是微软为何要开始针对Open source的因果涅盘.
  • Halloween III - 微软的反应, 令人冒火发毛. 本篇是所谓微软的Hype, Propaganda, 即夸大的文宣广告中的标准作品. 不知道微软在这方面功夫的人, 一定要看一看. 知道的人更要看一看, 学一学微软这种不要脸的出神入化功夫. (我怀疑有多少人有这么厚的脸皮.)
  • 万圣节文件的作者及相关人物背景调查 - linuxtoday.com, 外站. 万圣节文件迁连到微软内部相当多位高级主管, 及多位关键性研发人员/主管, 甚至有资深副总裁及两位微软政委会委员的出现. 该网页是对文件中有出现名字的人进行的背景调查. 微软之所以承认万圣节文件的原因, 据推测部份原因是因为这些人名均出现, 难以抵赖.
  • The Cathedral and the Bazaar : 教堂观与市集观, 作者为Eric S. Raymond, 这也许是开放原始码界最重要的文件了. Netscape因受本文影响而将原始码释出. (英文版)
  • Homesteading the Noosphere : 开拓智域疆土, 这是Eric S. Raymond另一篇非常著名的文章. 主要描写开放原始码社会文化现象, 这一篇比较不像"教堂观与市集观"那么出名 及受人喜欢, 因为作者将整个现象分析的十分精准, 身在其中者有许多人不喜欢被归类. 但我个人觉得这一篇是比"教堂观与市集观"更精彩的一篇杰作. (英文版)
  • FUD - 微软致胜法宝, 心理恐布战术
  • Shifting Standards(SS) - 规格污染战术
  • 反微软怨恨 - 这篇文章很道出译者的心声, 作者的一字一语都与译者心情相同(没他那么久, 那么强烈就是了), 因此将本篇短文翻译出来, 让有相同心情者一泄心中怨气. 另外, 您可以注意到, 在我所有的翻译文章中, 只要讲到"微软的人物", 都是用"它"或"它们", 其它的人物才会用"他/她们", 那, 不是笔误. 骂人的话我很会翻译, 本篇我翻译得很精准.
  • Microsoft Nightmare - 很有意思的图片, 用了好几项的合成处理技巧, 技术虽非完美, 但表达的淋漓尽致.

译者短评: 您可能会有所疑问, 什么是微软制作这份文件的起因呢?

根据译者由微软内部所收到的内幕消息(无法透漏消息来源), 主要原因, 因为IIS在网站伺服器市场上受到Apache的压制, 无法开拓. 微软的IIS是策略性产品, 免费附在Windows NT上, 因为IIS有许多的所谓"附加功能"(extension), 可进行"规格污染战术". 其次IIS只在NT上执行, 如果IIS占有市场率, Web-Lock可造成Windows NT 5.0(Windows 2000)在市场上的成功.

整个微软的计划推测大致如下: 

Windows 95/98 -> IE -> Frontpage ->
IIS Enable Web-Lock -> Intranet ->
Standard Pollution -> Windows NT ->
Encrypted Windows 2000 -> Proprietary Protocols ->
OS-Lock = 绝对的微软帝国统治

IE的作用在于进行Web-Lock的前置作业. 这才是真正为何Netscape vs IE这一战为何如此重要的原因. 据闻Windows 2000将执行码大量加密, 以防堵反向工程的进行.
